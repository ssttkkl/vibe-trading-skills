# 期权实战分析：从链上数据到策略决策

当用户问期权策略分析（"这个期权能买吗"、"卖什么期权好"、"能不能赌财报"）时，按此流程执行。

## 流程

### 1. 拉取期权链数据（yfinance）

```python
import yfinance as yf
mu = yf.Ticker("MU")
exps = mu.options               # 所有到期日
chain = mu.option_chain(exps[0]) # 最近到期
calls = chain.calls
puts = chain.puts
```

> 需要代理：`HTTPS_PROXY=http://127.0.0.1:7890`

### 2. 关键指标计算

每行期权必须输出以下字段：

| 字段 | 来源 | 含义 |
|------|------|------|
| `strike` | chain 原始 | 行权价 |
| `lastPrice` | chain 原始 | 最新成交价 |
| `volume` | chain 原始 | 成交量（流动性指标） |
| `openInterest` | chain 原始 | 未平仓量（深度指标） |
| `breakeven` | strike + premium (Call) | 盈亏平衡点 |
| `need%` | (breakeven/spot - 1) × 100 | 需要涨跌多少才赚钱 |
| `contract_cost` | premium × 100 | 1 张合约总成本 |
| 价态 | strike vs spot | 价内/价平/价外 |

价态判断：
```
Call: strike < spot - $5 = 价内, |strike-spot|<=5 = 价平, > spot + $5 = 价外
Put:  strike > spot + $5 = 价内, |strike-spot|<=5 = 价平, < spot - $5 = 价外
```

### 3. 流动性筛选

- **成交量 < 100**：流动性差，买卖价差可能很大，限价单建议不接
- **成交量 100-1,000**：一般，可以挂限价
- **成交量 > 1,000**：活跃，市价单也能接受
- **成交量 > 10,000**：流动性极好，几乎无滑点

### 4. 策略可行性检查（对接用户持仓）

**用户持仓约束——必须检查，比概率更重要：**

| 策略 | 要求 | 用户是否满足 |
|------|------|-------------|
| **Buy Call** | 只需现金付期权费 | ✅ 通常可以 |
| **Buy Put（对冲）** | 只需现金付期权费 | ✅ 通常可以。1张Put覆盖100股名义，注意用户持仓<100股时有过度对冲 |
| **Covered Call（卖出Call）** | **需持有 ≥100 股正股** | ❌ 用户一般只有几十股 |
| **Cash-Secured Put（卖出Put）** | 保证金 ≈ strike × 100 × 20% | ⚠️ 看用户现金够不够 |
| **Credit Spread** | 保证金 ≈ 宽度×100 | ✅ 保证金要求低，适合现金少的账户 |
| **Naked Call（裸卖Call）** | 最高权限 + 无限风险 | ❌ 永远不推荐 |

#### 合约规模不匹配
- 1 张 = 100 股名义本金
- 用户持 NVDA 45 股 → 买 1 张 Put 覆盖 100 股，多覆盖了 55 股（过度对冲）
- 用户持 MU 6 股 → 1 张 Put 覆盖 100 股，严重过度对冲
- 提示用户：期权合约规模远大于持仓时，考虑用正股加减仓替代

**例外：「便宜保险」模式**
- 当持仓足够大且 Put 足够便宜时，名义本金不匹配也可以接受
- 例：NVDA 45 股($9,481) ≈ 半张期权名义，$205 Put 仅 $55 → 92% 下行保护只花 $55，相当于 0.6% 的保费，值得买
- 判断标准：`put_premium / position_value < 1%` 且用户有强烈下行担忧 → 可以推荐
- 判断标准：`put_premium / position_value > 5%` 且持仓规模远小于 100 股 → 不如直接减仓

#### 财报期权必须提示 IV Crush + Theta 双杀
- 财报前 IV 极高 → Put/Call 都贵
- 财报后 IV 骤降 → 即使股价朝预期方向走，期权可能还是亏
- 到期前 3-5 天的 Theta 衰退极快
- **结论：持仓 6 股等小仓位，用正股赌财报比期权划算，不买末日轮**

### 5. 情景分析

对推荐的期权列出 3-5 个情景的 P&L：

```
NU @ $1,134 → $1,175 Call @ $64.16（平衡点 $1,239）

| 到期MU价 | 期权价值 | 期权盈亏 | 等效涨幅 |
|$1,150    | ~$0     | -$6,416  | -        |
|$1,175    | ~$10    | -$5,416  | +3.6%    |
|$1,200    | ~$30    | -$3,416  | +5.8%    |
|$1,239    | ~$64    | $0       | +9.3%    |  ← 平衡点
|$1,300    | ~$125   | +$6,084  | +14.6%   |
```

### 6. 常见陷阱（必须在结论中提示）

#### IV Crush（隐含波动率塌陷）
- 财报前 IV 极高 → 期权费贵
- 财报一出，无论涨跌 IV 大幅下跌 → 期权价值蒸发
- 经典剧本：MU 涨 2.6%，但 Call 还是亏钱（IV 从 120% 跌到 60%）

#### Theta Decay（时间损耗）
- 到期前 7 天内 Theta 加速衰减
- 买周度末日轮 ≈ 跟时间赛跑
- 溢价期权（价外）Theta 杀伤力最大

#### 合约规模不匹配
- 1 张 = 100 股名义
- 用户持仓 6 股 MU，期权对冲过度覆盖
- **优先建议正股加减仓（无时间损耗、无 IV Crush、规模灵活）**
