# Polymarket 深度调研 Prompt 模板

下面这份可以直接复制给 vibe-trading。它把**流程、方法论、判断标准**都合并在一起了，不再需要单独一章看方法论。

---

## 直接复制使用

### A. 单盘深度调研版

```text
语言：中文

你现在只做一件事：对下面这个 Polymarket 市场做“深度调研 + 尾盘套利判断”。

【市场输入】
- 市场标题：{{market_title}}
- slug / event id：{{slug_or_event_id}}
- 当前价格：Yes {{yes_price}} / No {{no_price}}
- 成交量：{{volume}}
- orderbook / 深度：{{orderbook_summary}}
- 规则原文：{{resolution_rules}}
- 页面评论摘要：{{comment_summary}}
- 相似盘 / 历史盘：{{similar_markets}}

【尾盘套利定义】
- 先用统一口径判断：只有当**规则客观**、**事件确定概率高**、**流动性足够可执行**、**尾部风险可控**时，才考虑叫“尾盘套利候选”。
- 市场是否“合理定价”不是核心问题；核心是**在事件确定概率上能否拿到足够的剩余套利空间**。
- 价格分档参考：
  - Yes/No 85-99.5¢：核心筛选区，优先看确定性和可执行性
  - 接近 100¢：先判断是否已经到“几乎没空间”的尾盘，再决定是否排除
  - 80-84¢：价值盘，弹性更大但不一定可执行
  - <80¢：投机盘，APY 可能很高但容易虚高
- 纪律性 checklist：
  - No 价格越高越确定
  - 到期越近越确定
  - 规则客观可量化（数据驱动 > 物理事实 > 主观判断）
  - 历史子市场全部判 No（同系列多个月验证）
  - 排除体育/名人/meme
  - 尾部风险可控

【调研目标】
1. 判断这个市场的规则是否清晰、客观、可结算。
2. 判断当前价格是否明显偏离你对真实概率的判断。
3. 判断流动性和深度是否足以支持交易，是否会有明显滑点。
4. 结合评论、新闻、历史相似盘、跨到期日价格差，判断有没有“送钱盘”机会。
5. 最后明确给出：
   - 是否值得下注 Yes / No
   - 哪一边更有正期望值
   - 是否属于“送钱盘”

【分析顺序】
### 1) 先看规则
- 找到 Resolution Rules 原文并复述关键条款。
- 检查时间窗、截止日、定义边界、排除条款。
- 确认裁决来源是官方、可信媒体共识，还是主观判断。
- 如果规则涉及“进入/访问/宣布/发布/达到阈值”，要明确什么算命中，什么不算。
- 规则越客观、越可量化，越适合做“送钱盘”研究；主观裁决盘优先降权。

### 2) 再看定价
- 直接对比市场价格与真实概率。
- 价格分档：
  - Yes/No 85-99.5¢：重点审查区，优先看确定性和可执行性
  - 接近 100¢：优先判断是否已经没有空间
  - No 80-84¢：更像价值盘，弹性大但风险也更大
  - No <80¢：更偏投机盘，APY 看起来高但未必可执行
- 如果市场价格已经很接近你估计的真实概率，说明 alpha 不大。
- 如果价格明显高于/低于你估计的概率，再进一步看是否是规则误读、尾部风险、或流动性错价。

### 3) 再看流动性与可执行性
- 先看成交量，再看 orderbook 深度和买卖价差。
- 粗略门槛：
  - >$1M：通常可认真考虑
  - $1K-$1M：只能算小资金机会，必须注意滑点
  - <$1K：通常不建议
- 如果价差很宽、深度很薄，就算方向对，也可能被滑点吃掉。
- 若市场看起来像“送钱盘”，先确认它不是“看起来便宜但根本吃不进去”。

### 4) 再看新闻、评论、历史盘
- 评论区只当情绪参考，不当证据。
- 先找最近的官方表态、可信新闻、公开行程、相关政策变化。
- 若有历史相似盘或同系列盘：
  - 看它们为什么判 Yes/No
  - 看这些判定原因现在是否仍然成立
  - 若同一事件有多个到期日，比较价格差，反推时间价值和当前状态
- 如果是地缘政治盘，要额外考虑尾部风险：台海、贸易战、制裁、军事升级、国内政治事件等。

### 5) 最后做送钱盘判断
- 只有当**规则客观**、**价格明显偏离**、**流动性足够**、**尾部风险可控**时，才把它当作潜在“送钱盘”。
- 重点找的是“市场把不可能/极低概率的事情定得太贵或太便宜”的盘。
- 如果只是高概率盘但价格已经合理，那就不是送钱盘，只是普通高胜率盘。

【输出格式】
请按下面结构输出：
1. 盘口概况
2. 规则确认
3. 定价判断
4. 流动性与可执行性
5. 评论/新闻/历史盘对照
6. 结论：
   - 是否送钱盘
   - Yes / No 哪边更有 EV
   - 建议仓位倾向（如果有）
   - 主要风险点

【写作要求】
- 只分析这个市场，不要扯别的持仓。
- 不要先入为主给结论，要先把规则和价格讲清楚。
- 如果证据不足，直接说不够，不要硬凑 alpha。
- 需要时可以引用相似市场、跨到期日价格差、或者订单簿深度来佐证。
- 中文输出，给完整分析，不要只给一句结论。
```

### B. 批量扫描并列对比版

```text
语言：中文

你现在只做一件事：对下面这一组 Polymarket 候选市场做并列深度调研，筛出最有 alpha / 最像“送钱盘”的盘。

【候选市场列表】
{{candidate_list}}

【共同背景】
- 本轮是批量扫描结果，不预设任何一个盘是主盘。
- 你必须把所有候选盘并列看待，逐个比较规则、定价、流动性、可执行性和尾部风险。
- 不要默认第一个盘更重要，也不要把某个盘当“主市场 + 其他补充”。

【调研目标】
1. 逐个判断每个候选盘的规则是否清晰、客观、可结算。
2. 逐个判断当前价格是否偏离你对真实概率的判断。
3. 逐个判断流动性、深度和滑点是否可接受。
4. 横向比较这些盘：
   - 哪些是真正的价值盘
   - 哪些只是高概率但已经定价合理
   - 哪些根本不可执行
   - 哪些受尾部风险影响最大
5. 最后输出一个排序：
   - 第一名最值得看 / 最像送钱盘
   - 备选盘
   - 应该直接排除的盘

【分析顺序】
### 1) 先把每个盘的规则拆开
- 对每个候选盘分别提炼 Resolution Rules 关键条款。
- 标出它的时间窗、截止日、定义边界、排除条款。
- 标出裁决来源是官方、可信媒体共识，还是主观判断。
- 规则越客观、越可量化的盘，优先级越高。

### 2) 再并列看定价
- 对每个候选盘，直接对比市场价格与真实概率。
- 重点识别：
  - 价格明显高于/低于真实概率的盘
  - 只是高概率但已经合理定价的盘
  - 价格看起来有 alpha、但实际上规则边界不清的盘
- 不要只看绝对价格，要看“价格偏离 + 规则质量”的组合。

### 3) 再并列看流动性与可执行性
- 对每个候选盘分别看成交量、orderbook 深度、买卖价差。
- 如果一个盘便宜但根本吃不进去，优先级要下调。
- 同等 alpha 下，优先选更容易执行、滑点更小的盘。

### 4) 再横向比较新闻 / 评论 / 历史盘
- 评论区只当情绪参考，不当证据。
- 先找最近的官方表态、可信新闻、公开行程、相关政策变化。
- 若有历史相似盘或同系列盘：
  - 看它们为什么判 Yes/No
  - 看这些判定原因现在是否仍然成立
  - 若同一事件有多个到期日，比较价格差，反推时间价值和当前状态
- 地缘政治盘额外考虑尾部风险：台海、贸易战、制裁、军事升级、国内政治事件等。

### 5) 最后做横向排序和筛选
- 只把同时满足以下条件的盘排前面：
  - 规则客观
  - 价格有明显偏离
  - 流动性足够
  - 尾部风险可控
- 如果只是高概率盘但价格已经合理，那就不是送钱盘，只是普通高胜率盘。
- 如果候选里没有明显 alpha，就直接说“没有值得下注的盘”。

【输出格式】
请按下面结构输出：
1. 候选盘总览
2. 逐盘分析（每个盘一小节）
3. 横向对比表
4. 排名
5. 结论：
   - 第一推荐盘
   - 次推荐盘
   - 排除盘
   - 哪些盘属于“送钱盘”候选
   - 主要风险点

【写作要求】
- 把这批盘当成同一组候选，必须并列比较。
- 不要默认任何单盘是主盘。
- 不要把候选列表写成“主盘 + 其他候选”。
- 如果证据不足，直接说不够，不要硬凑 alpha。
- 中文输出，给完整分析，不要只给一句结论。
```

---

## 使用说明

- 单盘分析：把 `{{market_title}}` 等字段替换成一个市场的数据即可。
- 批量扫描：把 `{{candidate_list}}` 填成筛出来的候选盘原始数据，让 vibe-trading **并列比较**，不是主盘 + 附属候选。
- 如果规则原文已经从 Gamma / 页面抓到，就直接填到 `{{resolution_rules}}`，不要让模型瞎猜。
- 如果只想问“这盘是不是送钱盘”，单盘版已经足够。
- 如果是扫描 → 过滤 → 调研 的工作流，应该用批量并列对比版。
